# 使用说明

## 基本使用

### 1. 上传 PDF

有两种方式上传文件：

- **拖拽上传**: 将 PDF 文件直接拖到页面中间的上传区域
- **点击选择**: 点击上传区域，在弹出的文件选择框中选择 PDF 文件

文件限制：
- 格式：仅支持 `.pdf` 文件
- 大小：最大 250MB
- 数量：单次仅支持 1 个文件

### 2. 设置目标大小

在"目标大小"输入框中输入你想要的文件大小（单位：MB）。

建议：
- 目标大小必须小于原文件大小
- 建议不要压缩过小，以免影响清晰度
- 通常压缩到原大小的 50-70% 质量损失不明显

示例：
- 原文件 50MB → 目标 20MB
- 原文件 100MB → 目标 50MB

### 3. 开始压缩

点击"开始压缩"按钮，系统会：

1. 上传文件到服务器
2. 分析 PDF 文件结构
3. 尝试多种压缩参数
4. 找到满足目标大小的最佳质量
5. 完成压缩

### 4. 下载结果

压缩完成后，点击"下载压缩文件"按钮即可下载。

下载的文件名会自动添加 `.compressed` 后缀，例如：
- 原文件：`报告.pdf`
- 压缩后：`报告.compressed.pdf`

## 压缩原理

### 压缩阶段

系统会按顺序尝试以下策略：

1. **轻度压缩**（0-60%进度）
   - 保持图片原始尺寸
   - 逐步降低 JPEG 质量
   - 压缩 PDF 对象流

2. **中度压缩**（60-85%进度）
   - 缩放图片尺寸
   - 进一步降低质量
   - 优化字体嵌入

3. **深度压缩**（85-100%进度）
   - 光栅化页面
   - 转为图片形式 PDF
   - 灰度化（可选）

### 质量优先

系统始终在满足目标大小的前提下，选择质量最高的方案。

## 常见场景

### 场景1：邮件发送

目标：将大 PDF 压缩到 20MB 以内（邮箱附件限制）

1. 上传 PDF
2. 目标大小输入：19
3. 开始压缩
4. 下载后作为附件发送

### 场景2：网盘节省空间

目标：将 PDF 库压缩以节省云盘空间

建议压缩比例：
- 图文混排 → 原大小的 60-70%
- 纯图片扫描版 → 原大小的 40-60%
- 纯文字 → 原大小的 80-90%

### 场景3：手机阅读

目标：压缩以便在手机上快速打开

目标大小建议：
- 100页以内 → 5-10MB
- 100-300页 → 10-20MB
- 300页以上 → 20-30MB

## 注意事项

### 什么情况下压缩效果不好？

- 已经高度压缩过的 PDF
- 纯文字且已优化的 PDF
- 包含大量矢量图形的 PDF

### 什么情况下可能产生空白页？

- 原 PDF 有加密
- 原 PDF 使用了特殊字体
- 原 PDF 结构损坏

遇到此问题请尝试其他压缩工具。

### 临时文件清理

- 上传的文件存储在系统临时目录
- 1小时后自动删除
- 重启服务器也会清理旧文件
